图论笔记

Posted on 2019-07-25

学习顺序

  • 无向图（简单连接）
  • 有向图（连接有方向）
  • 加权图（连接带有权值）
  • 加权有向图（连接即有方向性又带有权值）

无向图

  • 边：

    两个顶点之间的连接

  • 特殊情况：

    • 自环：一条连接一个顶点和其自身的边
    • 平行边：连接同一对顶点的两条边

  • 术语

    • 相邻顶点：

      通过一条边相连的两个顶点，并称该连接依附于这两个顶点

    • 子图：

      一幅图的所有的边的一个子集

    • 路径

      由边顺序连接的一系列顶点

    • 连通图：

      从任意一个顶点到另一个任意顶点都存在一条路径

      一幅非连通图是由若干的极大连通子图组成

表示无向图的数据类型

  • 图的基本操作api：

    • Graph(int V)：

      创建一个含有v个顶点但不含有边的图

    • Graph(In in)：

      从标准输入流in读入一幅图

    • int V()：

      顶点数

    • int E()：

      边数

    • void addEdge(int v, int w)：

      向图中添加一条边v->w

    • Iterable adj(int v)：

      和v相邻的所有顶点

    • String toString()：

      对象的字符串表示

  • 图的常用方法：

图的几种表示方法

  • 数据结构选择要求

    • 必须为可能在应用中碰到的各种类型的图预留出足够的空间
    • Graph的实例方法的实现一定要快——它们是开发处理图的各种用例的基础

  • 表示方法：

    • 邻接矩阵：

      使用V*V的布尔矩阵，两顶点间有边相连时bool为true。

      此表达方法不符合第一个条件，上百万个顶点的图所需的存储空间过大

    • 边的数组

      使用edge类，含有两个int实例变量（表示一条边及对应的顶点）

      此表达方法不符合第二个条件，实现adj（）方法（查询一个顶点的所有相邻顶点）需要检查所有的edge实例，即检查所有的边，成本过高

    • 邻接表数组

      使用以顶点为索引的列表数组，数组中的每个元素都为该顶点相邻的顶点列表

邻接表的数据结构

  • 数组+链表的数据结构
  • 要添加一条连接v，w的边，将w添加到v的邻接表，将v添加到w的邻接表
  • 性能特点
    • 使用的空间和V+E成正比
    • 添加一条边所需的时间为常数
    • 遍历顶点v的所有相邻顶点所需的时间和v的度数成正比（处理每个相邻顶点所需的时间为常数）

  • 实际应用中的操作：

    • 添加一个顶点
    • 删除一个顶点

    可通过扩展之前的API，使用符号表（ST）代替数组实现以上操作

    • 删除一条边
    • 检查图是否含有边v-w

    要实现以上操作，可能需要使用Set（集合）来实现邻接表，称此数据结构为邻接集

深度优先搜索

  • 参考走迷宫问题——Tremaux搜索

    • 选择一条没有标记过的通道，在你走过的路上铺一条绳子
    • 标记所有你第一次路过的路口和通道
    • 当来到一个标记过的路口时（用绳子）回退到上个路口
    • 当回退到的路口已没有可走的通道时继续回退

  • 图的深度优先搜索

    • 思路

      搜索一幅图只需用一个递归方法来遍历所有顶点

      在访问其中一个顶点时：

      • 将它标记为已访问
      • 递归的访问它的所有没有被标记过的邻居顶点

单向通道

当已经处理过依附于一个顶点的所有边时，我们就只能返回。

在无向图的深度优先搜索中，在碰到边v-w时，要么进行递归调用（w没有被标记过），要么跳过这条边。

第二次从另一个方向w-v遇到这条边时，总是会忽略它，因为v肯定已经被访问过了

广度优先搜索

  • 单点最短路径：

    给定一幅图和一个起点s，从s到给定目的顶点v是否存在一条路径？如果有，找出其中最短的那条。

  • 解决思路：

    找到从s到v的最短路径：

    1. 从s开始，在所有由一条边就可以到达的顶点中寻找x
    2. 第一层找完，如未找到x，递归向下

  • 实现

    使用队列保存所有已被标记但邻接表还未被检查过的顶点

    1. 将起点加入队列
    2. 取队列中的下一个顶点v并标记它
    3. 将与v相邻的所有未被标记过的顶点加入队列

深度优先算法和并查集比较

  • 理论上

    深度优先算法应该比并查集快，因为深度优先算法能保证所需的时间为常数而并查集不能

  • 实际上

    并查集是一种动态算法，不需要完整的构造并表示一幅图，所以并查集算法在实际应用中更快。

    深度优先搜索更适合实现图的抽象数据类型

符号图

  • 性质

    • 顶点名为字符串
    • 用指定的分隔符分隔顶点名
    • 每一行都表示一组边的集合，每一条边都连接着这一行的第一个名称表示的顶点和其他名称所表示的顶点
    • 顶点总数V和边的总数E都是隐式定义的

  • API

有向图

术语

  • 定义

    一幅有方向性的图（或有向图）是由一组顶点和一组有方向的边组成的，每条有方向的边都连接着有序的一对顶点

    一条有向边由第一个顶点指出并指向第二个顶点

  • 顶点的度数

    • 出度：该顶点指出的边的总数
    • 入读：指向该顶点的边的总数

  • 有向边的第一个顶点称为头，第二个顶点称为尾
