深入理解jvm虚拟机

主内存与工作内存

  • 变量：实例字段，静态字段和构成数组对象的元素
  • 主内存：虚拟机内存的一部分，可以和物理硬件主内存相互类比，用于存储java系统中所有的变量，负责线程间变量值的传递
  • 工作内存：线程私有内存（其他线程不可访问），保存线程使用的主内存中变量的副本拷贝，执行线程对变量的所有操作

线程的生命周期

  • Thread内部枚举类：State

    定义了线程的几种状态

    • new：新建线程对象，还没有调用start()方法
    • runnable：就绪，运行中统称为运行状态
    • block：阻塞于锁
    • waiting：等待其他线程作出一些特定动作（通知/中断）
    • timed_waiting：等待指定时长后自动返回
    • Terminated：线程执行完毕

  • 五种状态描述：

    • 新建
    • 就绪
    • 运行
    • 阻塞
    • 死亡

synchronized 和 lock的异同

  • 相同点：
    • 解决线程安全问题
  • 不同点：
    • lock锁显式调用，需要手动开启和关闭锁。synchronized锁是隐式调用，超出作用域（代码块，方法）自动释放
    • lock只有代码块锁，synchronized有代码块锁和方法锁
    • 使用lock锁，jvm将花费较少的时间来调度线程，性能更好，具有更好的扩展性
  • 优先顺序：
    1. lock
    2. 同步代码块（synchronized代码块）：已经进入方法体，分配了相应的资源
    3. 同步方法（synchronized方法）

线程的通讯

  • 同步方法

    • wait：一旦执行此方法，当前线程就进入阻塞状态，并释放同步监视器
    • notify：一旦执行此方法，就会唤醒被wait的一个线程，如果有多个线程被wait，唤醒优先级高的线程
    • notifyAll：一旦执行此方法，唤醒所有被wait的线程

  • 说明：

    1. wait，notify，notifyAll三个方法必须使用在同步代码块或同步方法中
    2. 调用者必须是同步代码块或者同步方法中的同步监视器（相同对象），否则会出现IllegalMonitorStateException异常
    3. java.lang.Object类中定义

  • 思考：

    • sleep和wait方法的异同
      1. 相同点：一旦执行方法，都可以使当前线程进入阻塞状态
      2. 不同点：
        1. 方法声明位置不同（Thread.sleep(静态方法)，Object.wait()）
        2. 调用范围/要求不同：sleep（）可以在任何需要的场景下调用，wait（）方法必须使用在同步方法/代码块中
        3. 关于是否释放同步监视器的问题：如果两个方法都使用在同步代码块或同步方法中，sleep方法不释放同步监视器（锁），wait方法会释放同步监视器（锁）

  • 经典例题：生产者/消费者问题

    • 生产者将产品交给店员，消费者从店员驱走产品
    • 店员一次只能持有固定数量的产品，如果生产者试图生产更多的产品，店员会叫停生产者
    • 如果店员没有产品，店员会叫停消费者

Runnable比较

  • 相比run方法，有返回值
  • 方法可以抛出异常
  • 支持泛型的返回值
  • 需要借助FutureTask类，比如获取返回结果

Future接口

  • 可以对具体Runnable，Callable任务的执行结果进行取消，查询是否完成，获取结果等
  • FutureTask事Future接口的唯一实现类
  • FutureTask同时实现了Runnable，Future接口，既可以作为Runnable被线程执行，又可以作为Future得到Callable的返回值

通过线程池创建多线程

  • 背景

    经常创建和销毁，使用量特别大的资源，比如并发情况下的线程，对性能影响很大

  • 思路

    提前创建好多个线程，放入线程池中，使用时直接获取，使用完放回池中。可以避免频繁创建销毁，实现重复利用。类似生活中的公共交通工具

  • 好处

    提高响应速度

    降低资源消耗

    便于线程管理
